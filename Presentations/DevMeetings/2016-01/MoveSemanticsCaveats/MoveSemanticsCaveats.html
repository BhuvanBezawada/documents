1. Dead Variables

std::move
  • is unspecified
  • variables which had std::move applied to it, should be treated as "dead"
Example: 

std::unique_ptr<Foo> foo1(new Foo);
...
std::unique_ptr<Foo> foo2 = std::move(foo);
...
foo1->doSomething(); // Error, foo1 is equal to nullptr

2. Special member function generation

  • move-constructor + move-assignment operator
  • rule of five
  • declaration of special member functions can obstruct generation of other special memeber functions
  • ...
Example: 

					Foo createFoo () {
  return Foo();
}
					

class Foo {
public:
  void doSomething() {...};
};

...
auto foo = createFoo(); // <- Move assignment
					

class Foo {
public:
  void doSomething() {...};
  ~Foo() {...}; // Want to do some logging
};
...
auto foo = createFoo(); // <- Copy assignment !!!
