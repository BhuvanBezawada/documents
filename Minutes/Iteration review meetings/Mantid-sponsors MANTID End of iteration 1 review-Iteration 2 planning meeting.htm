Aim of Iteration 1

To build the essential parts of the Mantid framework in order to be able to reproduce the same test that was performed on GAUDI and LOQ during the Pilot investigations.

Progress during Iteration 1

The first iteration of a project is always hard and difficult to predict, as the building of the software is often delayed by other essential tasks such as setting up the development and build environment and issues with third party libraries.

In our case the timing of the first iteration coincided with many of the team taking holidays or attending other work commitments away from ISIS.  As a result the majority of the work in this iteration has been done by Russell Taylor with help from Dickon, Matt and Laurent.

However after saying all of that it looks like we have achieved the aim of the iteration although we have had to make a few compromises, specifically:

·       The Logging framework chosen proved less mature than expected, and will have to be moved to iteration 2.

·       Work on the build environment started later than we would have hoped due to holidays.  It is working now, but more work is needed to integrate unit tests.

·       A few classes need unit tests expanding e.g. LoadRaw

·       The framework Manager API in Mantid only provides the minimum functions to complete the aim and needs completing.
